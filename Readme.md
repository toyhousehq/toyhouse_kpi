Toyhouse 人员绩效考核机制
======

为了调动团队成员工作积极性，保证任务目标的顺利达成，有效评估团队成员的贡献，特制定本绩效考核办法。

一、考核对象
----------------------------------
Toyhouse所有成员（组织所有人和财务除外）。

二、考核内容和方式
----------------------------------
（一）考核时间：每日考核，按周汇总。
（二）考核机制：每位Toyhouse成员具有100分基础分。完成任务进行加分，阶段性依据加分结果，给予比特币奖励或者现金、实物奖励。未完成任务或缺勤等行为进行减分，分数为0者失去成员资格。
（三）考核内容：成员每日工作完成情况及综合表现。

三、考核流程
----------------------------------
由制定工作计划、执行工作计划及工作考核三部分组成

(一)制定工作计划
1、项目负责人依据长期项目目标，进行任务划分；可使用Smartsheet。
2、成员依据项目分工，与负责人协调后制定每日个人工作计划，提前一天上传至Git，交与项目负责人审定；
3、负责人依据个人计划，制作工作安排表，上传至GitLab。

（二）执行工作任务
1、成员于第二日登录GitLab下载工作安排表后进行核对并签字后执行。

（三）进行绩效考核
1、依据任务完成情况进行分数统计；
2、每周对考核结果进行整合，并公布；
3、依据考核情况进行奖惩。

四、任务设置原则
----------------------------------
1、明确任务目标
2、明确任务时间
3、明确任务执行人
4、明确任务提交方式及要求
	  4.1 视频----时长、字幕、效果等
	  4.2 文档----字数
	  4.3 PPT----页数
	  4.4 图片----张数
	  4.5 数据----代码行数
	  4.6 记录报告（用于订饭、接待、开会等无直接数据产出的任务）
5、任务分值估计（基本以每1小时工作量为1分来估算，每人每日工作分值不宜超过15分）

五、评分细则
--------------------------------------
（一）加分机制
1、超预期完成任务：+150%（依据任务分值）
2、按要求完成任务：+100%（依据任务分值）
3、勉强完成任务：+50%（依据任务分值）

（二）扣分机制
1、无故迟到一小时：-1分。
2、无故缺勤半天：-5分
3、无故未完成任务：- 50%（依据任务分值）。

六、其他事项
（一）考核人员应坚持实事求是，客观公正地进行考核。被考核人认为考核结果严重不符合事实的，可以向法律部门提出申诉。经调查属实的，可给予纠正，并对相关责任人进行处理。
（二）本办法经批准后，于2014年7月起执行。
